# Collaboration Proposal Examples

This document provides examples of collaboration proposals to illustrate what makes a strong, focused proposal versus one that is too vague, unrealistic, or not well-aligned with project goals.

---

## Example 1: Strong Proposal ✅

### Title: Implement Wavelet Transform Analysis for RNA Secondary Structure

**Objectives:**
- Apply continuous wavelet transform (CWT) to RNA sequence data
- Identify periodic patterns in RNA secondary structures
- Create visualization tools for wave-based RNA analysis

**Alignment:**
- Directly addresses Wave-Based Analysis focus area
- Complements RNA Analysis objectives
- Enhances Data Visualization capabilities

**Approach:**
1. Research existing wavelet applications in bioinformatics
2. Implement CWT algorithm for RNA sequence data
3. Validate approach using benchmark RNA datasets
4. Develop interactive visualization dashboard
5. Document methodology and results

**Deliverables:**
1. Python library for wavelet-based RNA analysis (Month 2)
2. Validation report with benchmark comparisons (Month 3)
3. Interactive visualization tool (Month 4)
4. Technical documentation and tutorial (Month 4)

**Expertise:**
- PhD in Bioinformatics with focus on signal processing
- 5 years experience with wavelet analysis
- Published 3 papers on RNA structure prediction
- Proficient in Python, R, and D3.js

**Resources Needed:**
- From proposer: 8-10 hours/week for 4 months
- From project: Access to RNA datasets, feedback on approach
- Shared: Code review, testing, documentation

**Timeline:**
- Month 1: Research and algorithm design
- Month 2-3: Implementation and validation
- Month 4: Visualization and documentation

**Benefits:**
- Project gains novel analysis capability
- Proposer publishes methodology
- Advances field of RNA analysis

**Success Metrics:**
- Algorithm processes 1000 sequences/minute
- Identifies patterns with >90% accuracy vs. manual analysis
- Visualization tool has intuitive user interface

**Why This Works:**
- ✅ Specific, achievable goals
- ✅ Clear deliverables with dates
- ✅ Relevant expertise demonstrated
- ✅ Realistic timeline and resources
- ✅ Strong alignment with project focus
- ✅ Measurable success criteria

## Example 2: Needs Improvement 🔄

### Title: RNA Research Collaboration

**Objectives:**
- Work together on RNA stuff
- Make things better
- Publish papers

**Approach:**
We could look at RNA and analyze it using various methods.

**Timeline:**
Flexible, whenever works

**Expertise:**

## Example 3: Unrealistic Scope ❌

### Title: Universal RNA Analysis Platform

**Objectives:**
- Build complete RNA analysis platform
- Integrate all known RNA algorithms
- Create AI that predicts all RNA structures
- Revolutionize RNA research globally
- Win Nobel Prize

**Timeline:**
6 weeks

**Resources:**
Need access to all research data and unlimited compute

**Why This Doesn't Work:**
- ❌ Scope is far too broad
- ❌ Unrealistic timeline
- ❌ Overpromising on outcomes
- ❌ Resource requirements unreasonable
- ❌ Lacks focus and specificity
- ❌ Grand claims without evidence

**What Would Work Instead:**
- Focus on one specific aspect (e.g., secondary structure prediction)
- Realistic 6-12 month timeline
- Defined, achievable deliverables
- Reasonable resource requirements
- Measurable, practical goals

## Example 4: Poor Alignment ⏸️

### Title: Machine Learning for Stock Price Prediction

## Example 5: Strong But Deferred ⏸️

### Title: Machine Learning Classification of RNA Types

**Objectives:**
- Train deep learning model to classify RNA types
- Achieve >95% accuracy on benchmark datasets
- Provide API for real-time classification

**Deliverables:**
1. Trained model with validation report (Month 4)
2. REST API for RNA classification (Month 5)
3. Documentation and examples (Month 6)

**Why It's Strong:**
- ✅ Clear, specific goals
- ✅ Measurable success criteria
- ✅ Relevant to RNA analysis
- ✅ Demonstrated ML expertise

**Why It Might Be Deferred:**
- Project currently focused on wave-based analysis, not ML
- Team lacks capacity to support ML infrastructure
- Could be reconsidered when priorities shift

**Feedback:**
"This is a well-structured proposal with clear deliverables. However, our current focus is on wave-based signal processing approaches rather than machine learning. We'd encourage you to resubmit this proposal in Q3 2026 when we plan to explore ML techniques, or consider adapting the proposal to incorporate wave analysis methods."

## Example 7: Strong Incremental Proposal ✅

### Title: Add Unit Tests for RNA Parser Module

**Objectives:**
- Increase test coverage of RNA parser from 60% to >95%
- Add edge case handling for malformed sequences
- Document testing approach

**Deliverables:**
1. 50+ unit tests covering core parsing functions (Week 2)
2. Integration tests for file formats (Week 3)
3. Test documentation and examples (Week 4)

**Expertise:**
- Software engineer with 3 years Python testing experience
- Contributor to BioPython testing suite
- Familiar with pytest, unittest, and mocking

**Time Commitment:**
5 hours/week for 4 weeks (20 hours total)

**Why This Works:**
- ✅ Small, focused scope
- ✅ Clear, achievable goal
- ✅ Concrete deliverables
- ✅ Relevant experience
- ✅ Realistic time commitment
- ✅ Valuable contribution

**Note:** Not all collaborations need to be large-scale. Focused, incremental improvements are valuable!

## Key Takeaways

### Do's ✅
- Be specific about objectives and deliverables
- Demonstrate relevant expertise
- Propose realistic timelines
- Show clear alignment with project
- Define measurable success
- Commit realistic resources

### Don'ts ❌
- Make vague or grandiose claims
- Overcommit or underestimate effort
- Propose work outside project scope
- Assume without evidence
- Lack concrete details
- Ignore resource constraints

### Remember
- Quality over quantity
- Focused beats broad
- Evidence beats speculation
- Incremental progress beats grand visions
- Clear communication builds trust
